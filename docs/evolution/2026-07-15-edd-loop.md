# Evolution — edd-loop (`/phil:edd`)

**Finalized:** 2026-07-15 · **Branch:** `feature/edd-loop` · **Type:** developer-facing plugin feature (prose skill + agent)
**Waves run:** DISCUSS → DESIGN → DISTILL → DELIVER (DEVOPS skipped — prose tool, no CI/infra; no SPIKE)

## Feature summary

A new `/phil:edd` command + `skills/edd/SKILL.md` + `agents/edd-evidence-producer.md` — an
**Expectation-Driven Development front-door**. It captures a developer's intent as discrete
**expectations**, classifies each, and — biased toward getting out of the way — **off-ramps** to
nwave (user-facing) or `phil:work` (invisible) whenever those engines' native oracles already prove
the intent. Only for the **qualitative residue** (expectations no test can cheaply assert —
"errors are helpful", "the API feels ergonomic") does it delegate the build to one engine and attach
a **scaled executed-evidence gate** the developer adjudicates. It is the expectation-driven sibling
to `phil:work`: frame → delegate the build → inherit the engine's oracle → add only the one gate the
engine lacks → leave a trail.

## Business context (JTBD)

`job_id: prove-qualitative-expectations-with-evidence` — persona **Avery**, a developer who delegates
building to AI agents and evaluates results against intent (the "author → evaluator" shift). The
crux: AI produces code faster than line-by-line review scales; the same AI that builds also
"verifies"; and qualitative intent gets dropped by formal tests or accepted on narration. `/phil:edd`
supplies a lightweight expectation → executed-evidence → human-adjudication loop that **reuses**
nwave/`phil:work` for everything checkable and adds a gate **only** for the qualitative residue —
the missing third option for the uncheckable goals `phil:work` FRAME currently refuses.

## Verification

No CI collector (prose skill). The `skills/edd/self-test/` harness is the acceptance + regression
gate, driven by human/model exactly like `phil:work`/`refactor-tests`. All 7 fixtures reach their
`expected.md` decision (`OFF-RAMP` / `CLASSIFY-CHECKABLE` / `GATE-POINT-EXISTING` / `GATE-COMMISSION`
/ `REJECT-NARRATION` / `BLOCK-DONE` / `DOCUMENT-TRAIL`). The two silent-failure guards are pinned:
**producer ≠ builder** (structural — the producer agent has no Edit/Write) and **narration ≠
evidence**. Whenever `SKILL.md`, `commands/edd.md`, or `agents/edd-evidence-producer.md` changes,
re-drive the fixtures. Mutation testing: N/A (no code unit tests).
