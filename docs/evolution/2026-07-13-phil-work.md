# Evolution — phil-work (`/phil:work`)

**Finalized:** 2026-07-13 · **Branch:** `docs/phil-work-brief` · **Type:** developer-facing plugin feature (prose skill / orchestrator)
**Waves run:** DISCUSS → DESIGN → DISTILL → DELIVER (DEVOPS skipped — prose tool, no CI/infra; no SPIKE)

## Feature summary

A new `/phil:work` command + `skills/work/SKILL.md` — a **wave-based orchestrator for invisible
(non-user-facing) technical initiatives**: refactoring, re-architecting, cleanup, migrations,
dependency/perf work. It is the invisible-work sibling to nwave. It is a **general contractor**: it
frames, plans, safeguards, sequences, and documents — then **delegates execution to the tactical
skills already in the plugin**, inheriting each delegate's gate rather than building its own.

## Business context (JTBD)

`job_id: deliver-invisible-work-with-discipline` — persona **Quinn**, a codebase steward, wants the
same disciplined, test-backed, documented workflow nwave gives user-facing features, but for work
the user never sees — without fighting nwave's user-story frame or doing it ad hoc with one-off
skills. The crux: invisible work has no persona/journey/story, and the tactical skills are
single-purpose (none discuss, plan across waves, or document). `/phil:work` supplies that
connective tissue with a two-gate spine: **preservation floor** (always) + **declared initiative
goal** (per initiative).

  until an initiative needs it.
