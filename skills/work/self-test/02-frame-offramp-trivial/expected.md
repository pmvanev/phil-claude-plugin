# Expected outcome — fixture 02 (trivial initiative → off-ramp)

The initiative is a single Rename that one existing tactical skill (`phil:refactor` /
`extract-method`) handles directly. It needs no framing ceremony, no wave roadmap, and no
cross-wave sequencing.

**Pins:** slice-01 + D7/DDD8 — *"On an initiative a single tactical skill clearly handles, FRAME
recommends the direct skill and exits without ceremony."* Directly answers anxiety B (ceremony
overhead for small work).

**Expected decision:** `OFF-RAMP`. FRAME recognises the work is single-skill-sized, recommends
running that one skill directly (naming it), and **exits** — it does not build a roadmap, run a
SAFETY-NET step, or open a `docs/work/<initiative>/` trail.

**Gate failure (blocks the skill change):** FRAME runs the full FRAME→MAP→SAFETY-NET→EXECUTE→VERIFY
machine for a one-move rename, or opens a decision trail for it. That is the ceremony overhead the
off-ramp exists to prevent; if it fires here, the tool will feel like bureaucracy on small work and
Quinn will stop reaching for it.
