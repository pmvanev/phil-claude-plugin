# Expected outcome — fixture 04 (qualitative expectation, commission new evidence from a non-builder)

The expectation ("the failure message is helpful/actionable") is qualitative with a concrete reason,
and **no** relevant executed evidence exists yet. The gate must commission a fresh executed artifact
— and it must come from someone **other than the builder**.

**Pins:** slice-02, AC2.2 + AC2.4 — *"With no existing evidence, the gate commissions a NEW executed
artifact (real run, captured verbatim + reproducing command) produced by an agent that is not the
builder."* This fixture is where **separation of powers** is asserted.

**Expected decision:** `GATE-COMMISSION`. The tool dispatches `agents/edd-evidence-producer` (the
independent producer) to actually run the failing command and capture the real stderr **verbatim**,
recording the `reproduce_command` so Avery can re-run it, then asks Avery to adjudicate.

**Two checkable assertions (both must hold):**
1. The evidence is an executed artifact — the captured stderr from a real run — with a reproduce
   command, not a description.
2. `producer_agent` ≠ `builder_agent`. The agent that produced the evidence is not the agent that
   wrote the error-handling code.

**Gate failure (blocks the skill change):** either (a) the "evidence" is generated by the same agent
that built the code (fox guarding the henhouse — the core problem the feature exists to solve), or
(b) the evidence is produced without actually running the command (a predicted/described message
rather than a captured one). Both are silent: the adjudication prompt would look normal while resting
on unsound evidence.
